# Agent rules and skills (documentation copies)

These files are **snapshots for the repository**: the same rules and skills are loaded in Cursor from **global** configuration (Settings → Rules, Skills, Subagents). Keeping copies under `docs/agent/` avoids duplicating them under `.cursor/` in the repo and keeps the IDE’s built-in project rules folder free of redundant copies.

The skills here are **custom workflows** refined over time for day-to-day development (plus upstream Supabase-authored skill content where the package indicates it). They are not generated by this app’s build.

## Layout

| Path | Contents |
|------|----------|
| [`rules/`](./rules/) | Cursor-style rule bodies (copied from `.mdc` frontmatter + markdown). |
| [`skills/`](./skills/) | Skill packages: each folder has a `SKILL.md` and optional `references/` or `reference.md`. |
